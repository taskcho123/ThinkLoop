# ThinkLoop

> **Code with AI. Think like a Developer.**
> https://thinkloop-1706c.web.app/

## 📌 프로젝트 소개

ThinkLoop는 AI를 사용하는 개발자를 위한 **Daily Reflection Workspace**입니다.

AI는 빠르게 코드를 작성해 주지만, 프로젝트의 방향성, 사용자 경험(UX), 유지보수성, 확장성, 그리고 왜 이런 구조를 선택했는지는 결국 개발자가 고민해야 합니다.

ThinkLoop는 이러한 **'생각하는 시간'**을 습관으로 만들기 위해 제작된 웹 서비스입니다.

프로젝트를 관리하는 것이 목적이 아니라, AI 시대에도 개발자의 사고력과 기록 습관을 유지하도록 돕는 것이 목표입니다.

# 🤔 문제 정의

최근 개발 방식은 다음과 같은 흐름으로 이루어지는 경우가 많습니다.

```
문제 발생
    ↓
AI에게 질문
    ↓
코드 생성
    ↓
복사 및 적용
    ↓
다음 문제 발생
```

이 과정은 개발 속도를 높여주지만 다음과 같은 문제를 만들 수 있습니다.

* 프로젝트 전체 흐름을 이해하지 못한 채 개발하게 된다.
* "왜 이렇게 구현했는지"를 시간이 지나면 설명하기 어려워진다.
* 사용자 경험(UX)이나 유지보수성보다 당장의 구현에 집중하게 된다.
* AI가 작성한 코드와 내가 고민한 부분의 경계가 흐려진다.
* 프로젝트 회고가 부족하여 성장 과정이 기록되지 않는다.

즉, **코드는 AI가 작성하지만, 사고까지 AI에 의존하게 되는 문제**가 발생할 수 있습니다.

# 🎯 목표

ThinkLoop는 AI 사용을 줄이기 위한 서비스가 아닙니다.

AI를 적극적으로 활용하되, 매일 잠시 멈춰 프로젝트를 돌아보고 스스로 생각하는 습관을 만드는 것을 목표로 합니다.

사용자는 매일 다음과 같은 질문에 답하며 프로젝트를 기록합니다.

* 오늘 AI는 무엇을 해주었는가?
* 나는 어떤 결정을 내렸는가?
* 어떤 부분을 개선했는가?
* 오늘 얻은 인사이트는 무엇인가?
* 내 프로젝트에 어떻게 적용할 수 있을까?

이러한 기록은 시간이 지나면 프로젝트 회고와 포트폴리오의 기반이 됩니다.

# ✨ 핵심 기능

## 📖 개발 인사이트

매일 기술 블로그를 읽고 기록합니다.

* 읽은 글 요약
* 느낀 점
* 현재 프로젝트에 적용할 아이디어

단순히 정보를 소비하는 것이 아니라 자신의 프로젝트와 연결하여 사고하도록 돕습니다.

---

## 🚀 프로젝트 관리

현재 진행 중인 프로젝트를 관리합니다.

* 프로젝트 생성
* 진행 상태 관리 (계획 / 진행 / 종료)
* 프로젝트별 회고 기록

---

## 📝 Daily Reflection

매일 프로젝트를 되돌아보며 기록합니다.

* AI가 오늘 한 일
* 내가 직접 고민하고 결정한 부분
* 개선된 점
* 회고
* 다음 목표

AI의 기여와 개발자의 사고 과정을 분리하여 기록할 수 있습니다.

---

## 📈 성장 기록

모든 기록은 프로젝트별 타임라인으로 저장됩니다.

이를 통해

* 프로젝트 진행 과정
* 의사결정 과정
* 개선 과정
* 성장 과정

을 한눈에 확인할 수 있습니다.

# 💡 ThinkLoop가 추구하는 가치

ThinkLoop는 생산성을 높이는 서비스가 아닙니다.

생산성보다 **사고력(Thinking)**을 중요하게 생각합니다.

AI는 코드를 작성할 수 있지만,

* 어떤 문제를 해결해야 하는지
* 사용자에게 어떤 경험을 줄 것인지
* 앞으로 어떻게 확장할 것인지

를 결정하는 것은 개발자의 역할입니다.

ThinkLoop는 그 과정을 매일 기록하고 돌아볼 수 있는 공간입니다.
